# Касания: концепция и работа

> Короткий how-to по тому, что такое «касание» в «Штабе», зачем оно нужно и как правильно его использовать в пятничном ритуале.

## Что это

**Касание** — любое подтверждённое взаимодействие тренера с клиентом: сообщение, звонок, тренировка, что-то ещё. Фиксируется датой (`touched_at = CURRENT_DATE`) и опциональной заметкой.

В БД это таблица `touches`: `{client_id, trainer_id, type, touched_at, note}`. Один клиент — много касаний, по убыванию даты.

## Зачем

1. **Триггеры на `/today`** считают «дни без касания» (`lastTouchDate = GREATEST(last_session_date, created_at, MAX(touches.touched_at))`). Без касаний клиент молча уплывает в «остывших», потом в «тихих» — и пропадает с радара.
2. **История касаний на `/clients/[id]`** даёт картину динамики отношений — кому когда и что писал.
3. **Экспорт для Claude** собирает строки только по тем, у кого триггер активен. Отметил касание — клиент выпадает из экспорта на этой неделе. Это и есть смысл: Claude не пишет тем, кому уже написали.

## Как отмечать

### Способ 1 — `/today`: «✓ Отметил»

В один тап. Тип = `message`, без заметки. Используй когда:
- Только что написал клиенту в мессенджер
- Не нужно фиксировать детали
- Хочешь убрать строку из списка триггеров

Строка пропадает мгновенно (`useOptimistic`). Если сервер вернёт ошибку — тост, строка вернётся.

### Способ 2 — `/clients/[id]`: «Отметить касание»

Открывает модалку: `тип` (Сообщение/Звонок/Тренировка/Другое) + `заметка` (до 500 символов). Используй когда:
- Это звонок или тренировка — а не сообщение
- Нужно зафиксировать договорённость («четверг 18:00»)
- Дополняешь историю задним числом — например, после телефона

## Что меняется после касания

1. В `touches` появляется запись с `touched_at = CURRENT_DATE`.
2. `listClientsWithLastTouch` пересчитывает `lastTouchDate` — берёт `MAX` из всех источников.
3. На `/today` триггер этого клиента пропадает (если был) — `daysSince` обнуляется.
4. На карточке клиента сразу видна новая запись истории.
5. `revalidatePath('/today')` и `revalidatePath('/clients/${id}')` сбрасывают кэш — данные актуальные.

## Пятничный ритуал (для чего это всё)

1. Открыть `/today`. Видим список «кому пора написать», сгруппирован по приоритету (🔇 тихий → 🔴 срочно → 🟠 средне → 🟡 можно подождать).
2. Прочитать каждую строку, открыть карточки тех, у кого слабая память — освежиться.
3. **Выбрать** галочками тех, кому реально буду писать сейчас (не всех — только тех, кого собираешься обработать в эту сессию).
4. Кликнуть **«Скопировать для Claude (N)»** — текст с шаблоном-промптом и строками клиентов уходит в буфер. **Контактов в нём нет — это инвариант** (тип `ClientForExport` физически без `contact`).
5. Вставить в чат Claude → получить N черновиков → вычитать → отправить из мессенджера.
6. По каждому отправленному — на `/today` тапнуть **«✓ Отметил»**. Строка пропадает.
7. Если был звонок или тренировка вместо сообщения — открыть карточку клиента, нажать **«Отметить касание»**, выбрать правильный тип, опционально записать заметку («четверг 18:00»).

## Модалка «Без личного факта»

Если в выборе есть клиенты без `personal_fact` — модалка предупредит. Личный факт — это якорь, без которого черновик звучит шаблонно («тренируется к свадьбе сестры», «бегает марафоны»). Две опции:

- **«Допишу факт»** — закрыть модалку, открыть карточку клиента по ссылке, добавить факт, потом снова выбрать и скопировать.
- **«Скопировать без них»** — собрать экспорт только из тех, у кого факт есть. Тогда клиенты без факта останутся в триггерах до следующей сессии.

## Чего избегать

- **Не отмечать «впрок».** Касание = состоявшееся событие. Если напишешь завтра — отметь завтра.
- **Не сливать события.** Если был и звонок, и сообщение — две отдельные записи.
- **Не путать с `last_session_date`** в карточке клиента: то — дата последней **тренировки**. Касание шире — любое взаимодействие.
- **Заметки кратко.** Это для тебя через неделю — не для Claude и не для отчёта.

## Удалённый клиент

На карточке soft-deleted клиента кнопки «Отметить касание» нет, история скрыта. `recordTouch` такого клиента вернёт ошибку «Клиент не найден». Это намеренно — касания имеют смысл только для активных клиентов.

## Sanity-проверка

В dev-режиме доступна страница `/dev/export-sanity` — 5 кейсов гоняют `buildClaudeExport` и проверяют:
- Текст собирается из клиентов с фактом
- Клиенты без факта попадают в `missing`
- В тексте нет `@` и нет `+7` — контакт не утекает

После любой правки `lib/export/claude.ts` — открыть эту страницу, убедиться `5 / 5 прошло`.
