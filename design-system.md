# Дизайн-система: «Штаб» — админка тренера

Источник истины — **брендбук «Тренер у дома» v2.0** (токены §14). Этот документ не изобретает стиль, а адаптирует его с лендинга (рассказ) под админку (данные). Принцип адаптации: палитра, шрифты, стекло и движение — без изменений; плотность и радиусы — компактнее, это рабочий инструмент, а не витрина.

Настроение: спокойный · собранный · тёмный неон. Тот же «тихий premium» — админкой пользуется сам владелец бренда каждый день, она не имеет права выглядеть чужой к лендингу.

## Палитра (из брендбука, 1:1)

| Роль | HEX | Использование в админке |
|---|---|---|
| Акцент / primary CTA | `#2CE6FF` cyan | одна главная кнопка на экран («Скопировать для Claude»), ссылки, активные элементы, фокус |
| Вторичный акцент | `#8B5CFF` violet | градиенты фона, выделение «сегодня» |
| Профиль «здоровье» | `#36FF9D` green | бейдж профиля; также success-состояния |
| Профиль «форма» | `#FF4FD8` pink | бейдж профиля |
| Профиль «энергия» | `#2CE6FF` cyan | бейдж профиля |
| Предупреждение / высокий триггер | `#FF9F43` orange | 🔴/🟠 триггеры, ошибки валидации, плашка «нет личного факта» |
| Фон базовый | `#0E1117` | под градиентом, как на лендинге |
| Подложка карточек | `#151923` (`glass #141925`) | стеклянные карточки |
| Вложенные элементы | `#1B2030` | строки таблиц, инпуты |
| Текст | `#E8ECF4` / `#9AA3B5` / `#5E6678` | основной / вторичный / подписи |
| Линии | `rgba(255,255,255,.09)` / `.055` | бордеры карточек / hairline-разделители |

Семантика триггеров (в брендбуке нет красного — не вводим): высокий приоритет = orange + glow, средний = violet, низкий/инфо = `tx2`. Эмодзи триггеров (🔴🟠🟡🔇) сохраняем — они уже несут цвет и переживут любой рестайлинг.

## Типографика (из брендбука, 1:1)

| Уровень | Шрифт | Размер/высота | Где в админке |
|---|---|---|---|
| H1 (заголовок экрана) | Gerhaus, КАПС | 24–27px | «Сегодня», «База», «Панель» — по одному на экран |
| H2 (секция) | Gerhaus, КАПС | 18–20px | блоки внутри экрана |
| Body | Nunito (subset cyrillic) | 15/1.5 | весь контент; на лендинге 16 — в админке 15 ради плотности |
| Small / подписи | Nunito | 13/1.4 | вторичные подписи |
| Метки, цифры, триггеры, счётчики | JetBrains Mono 500 | 12–13, трекинг +1.6 | kicker-метки, значения панели, даты, бейджи |

Запреты брендбука действуют: body капсом нельзя, эмодзи как иконки нельзя (иконки — Lucide; эмодзи только в значениях триггеров как данные), смешивать дисплейные шрифты нельзя.

## Сетка и отступы

База 4px. Шкала: 4·8·12·16·24·32·48. Контейнер админки: 1080px (как лендинг), таблица базы может уходить в full-width на десктопе. Брейкпоинты mobile-first: 360 / 768 / 1080. Межсекционный отступ: 32–48px (на лендинге 64–88 — админка плотнее сознательно).

## Радиусы и стекло (адаптация)

Лендинговые 30–42px на рабочих элементах громоздки. Шкала админки: `r-sm 12` (инпуты, строки, бейджи) · `r-md 16` (кнопки, модалки) · `r-lg 22` (карточки, = брендбук `radius-md`). Стекло — рецепт брендбука без изменений: подложка `#141925` + светлый градиент + бордер `rgba(255,255,255,.09)` + edge-highlight `inset 0 1px 0 rgba(255,255,255,.10)` + многослойная тень. Hairline-разделители внутри карточек: `rgba(255,255,255,.055)`.

Правило рамок §06 в админке: всё интерактивное — виджет → в стекле с бордером. Заголовок экрана и пустые состояния — рассказ → на градиентном фоне без рамки.

## Компоненты

| Компонент | Варианты | Состояния |
|---|---|---|
| Кнопка | primary (cyan + glow + кольцо) / secondary (стекло + бордер) / ghost | default · hover (подъём −2px, glow+) · active · disabled · loading (спиннер, ширина не прыгает) |
| Поле ввода | text · select · date · textarea | default · focus (cyan-кольцо) · error (orange бордер + текст под полем) · disabled |
| Бейдж | профиль (green/pink/cyan, точка+текст) · статус (нейтральный) · триггер (mono + цвет приоритета) | — |
| Карточка клиента | drawer (≥768) / полный экран (<768) | просмотр · редактирование · плашка «нет личного факта» (orange) |
| Таблица базы | desktop: таблица · mobile: список карточек | строка hover `#1B2030` · выбранная (чекбокс) — cyan-кольцо слева |
| Тост | success (green) · error (orange) · info | автоскрытие 4s; «Скопировано. Вставляй в Claude» |
| Модалка | подтверждение («Отметить ушедшим?») | одна кнопка primary, вторая ghost |

Правила: **один cyan-CTA на экран** (на «Сегодня» это «Скопировать для Claude» — даже «+ Лид» там secondary) · размеры кнопок из брендбука: LG 16/30·17 / MD 13/24·15 / SM 10/18·13 · никаких трёхцветных градиентов и кнопок-простыней.

## Движение (из брендбука, 1:1)

Появление: fade + translateY(24→0), `cubic-bezier(.2,.7,.2,1)`, 0.6s; stagger в списках 90ms. Hover — только transform/opacity (GPU). Никакого scroll-jacking; `prefers-reduced-motion` отключает всё. Скелетоны вместо спиннеров на загрузке списков.

## Пустые состояния и ошибки (голос §11: прямо, тепло, без сюсюканья)

| Где | Так говорим | Так — нет |
|---|---|---|
| «Сегодня» пустой | «Триггеров нет. База под контролем — так держать.» | «Ура! Всё супер! 🎉» |
| База пустая | «База пустая. Импортируй таблицу или добавь первого клиента.» + 2 CTA | «Здесь пока ничего нет :(» |
| Ошибка сохранения | «Не сохранилось. Проверь соединение и попробуй ещё раз.» | «Произошла непредвиденная ошибка #5042» |
| Импорт с ошибками | «12 строк загружено, 2 с ошибками: строка 7 — дата, строка 9 — статус. Поправь и перезалей файл.» | «Ошибка импорта» |
| Нет личного факта | «Без личного факта сообщение не соберётся. Допиши — и клиент попадёт в экспорт.» | молча исключить из экспорта |

## Tailwind: каркас конфига для шага M

```ts
// tailwind.config.ts — theme.extend
colors: {
  bg: { DEFAULT: '#0E1117', 2: '#151923', 3: '#1B2030', glass: '#141925' },
  tx: { DEFAULT: '#E8ECF4', 2: '#9AA3B5', 3: '#5E6678' },
  cyan: '#2CE6FF', violet: '#8B5CFF', pink: '#FF4FD8',
  green: '#36FF9D', orange: '#FF9F43', blue: '#4D7DFF',
  line: 'rgba(255,255,255,.09)', 'line-soft': 'rgba(255,255,255,.055)',
},
borderRadius: { sm: '12px', md: '16px', lg: '22px' },
fontFamily: {
  display: ['Gerhaus', 'Georgia', 'serif'],
  sans: ['Nunito', 'system-ui', 'sans-serif'],
  mono: ['"JetBrains Mono"', 'monospace'],
},
transitionTimingFunction: { soft: 'cubic-bezier(.2,.7,.2,1)' },
boxShadow: {
  glow: '0 0 24px rgba(44,230,255,.35)',
  'glow-orange': '0 0 20px rgba(255,159,67,.30)',
},
```

Чек перед сдачей экрана (из свода §13): единый контейнер · градиентный фон, не плоский · стекло и hairline · один cyan-CTA · Gerhaus-КАПС только в заголовках · Lucide-иконки · плавное появление · честный интерфейс — никаких фейковых статусов и цифр.
